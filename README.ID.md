# Manajemen Hubungan Pelanggan _(atau Customer Relations Management disingkat CRM)_
Mengola Daftar Nama pelanggan terkadang bisa berbeda tujuannya, terkadang profil pelanggan bisa memiliki lebih dari 2 perusahaan, dan seringkali memiliki nomor telpon dan alamat yang berbeda - beda.
Terkadang pada beberapa aplikasi, CRM masih belum sepenuhnya mencapai keperluan user terutama salesman atau business owner yang cukup berpengalaman dalam memprofilkan calon prospek mereka.
Untuk itulah, sebagai seorang programmer sekaligus mantan salesperson dengan pengalaman kerja 5 tahun didunia penjualan, Repositori CRM_DN _(disingkat 'Daftar Nama Customer Relations Management')_ didirikan pada Github sebagai OS untuk bisa saling berkolaborasi dalam pembuatan project CRM ini.

## Rencana Pembangunan CRM ini:
1. Pembuatan struktur dan Relasi Tabel MySQL RDBMS yang membuat CRM saling terintegrasi satu dengan lain
2. Prototype Admin Panel sebagai Aplikasi Utama CRM yang dirancang dan dikhususkan bagi salesperson maupun pelaku usaha dalam mengola CRM yang memenuhi kebutuhannya
3. Pembuatan Server menggunakan PHP atau Laravel dalam memanggil Database ke Server untuk dibawa ke Panel Admin
