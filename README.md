# Customer Relations Management (CRM)
Managing customer name list sometimes have different goals, sometimes customers are unique, where they might own more than 2 companies alone, and most of them owns different phone numbers and Address, like in most Asian Countries.
Sometimes on different application, CRM still haven't fulfill their user's demand outmostly salesperson or business owner that have experience profiling their potential customers.
As a programmer and a former salesperson with 5 years experience, I create a CRM_DN Repository _(abbreviated as 'Customer Relations Management (CRM) Integrated Contact Listing')_ as an OS that can colaborate in this CRM Project.

## Planning in Building this CRM:
1. Structuring and creating the MySQL RDBMS Tables to make CRM integrated with one another.
2. Admin panel prototype as main CRM App designed for salesperson and business owner in managing CRM that fulfill their needs.
3. Create a Server using Laravel by calling the Database via Server and send to the Admin Panel.
